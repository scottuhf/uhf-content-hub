Universal Rules

Non-negotiable standards that apply to every video regardless of style. These exist because the same mistakes kept appearing across rounds. Know them before you open the project.

01
Hook First — 0:00–0:04 Is Your Most Important Work
The first 2–4 seconds need to be your best work every time. 90% of the rest matters a whole lot less. If the hook is weak, nothing else saves the video. Spend disproportionate effort here.
02
Text Color = Meaning — Never Decoration
Color only appears when it means something. Never use it just to have it. Never default to UHF pink or blue in captions or body text.
🟢 Green = Good / Positive 🔴 Red = Bad / Negative 🟡 Yellow/Orange = Cautious / Middle Ground ⬜ White = Default / Neutral
03
CAPS = Emphasis Only — Not the Default
80% of any video should be lowercase white text. All caps is reserved for words or phrases that require emphasis. Using caps everywhere makes nothing feel emphasized — it kills the effect. This rule was violated repeatedly across multiple rounds.
04
No Text Outline — Use Shadow Instead
When text needs contrast from a background, use a drop shadow, not an outline/stroke. Outlines look cheap and were flagged from Round 1 of Style 1.
05
Always Have Movement — Nothing Fully Static
At all times, something on screen should be animating or moving. Even subtle element drift or entrance animations count. A fully static frame reads as dead. Elements animate in/out; the screen is never frozen.
06
Sound Design Is Part of the Edit — Not an Afterthought
SFX, impacts, whooshes, and thematic audio are integral to the style, not added at the end. Think about sound from the start of the edit. Each style has its own SFX approach — see individual style rules.
07
Camera Pans the Scene — Elements Don't Shrink
To create depth or a pan-out effect, the camera moves — not the graphics scaling down. The whole scene pans together. Shrinking individual elements does not replicate this and looks wrong. A reference video (Pan out.mp4) was sent for Style 3.
08
Real Headshots Over Animated Clip Art (Style 2+)
Use actual player headshots — not animated clip art characters or illustrated figures. This applies to Style 2 and Style 3. Round 1 of Style 2 flagged this immediately.
09
Read Revisions Closely — Repeat Errors Are Tracked
UHF colors in text, wrong caps usage, and outline vs shadow were flagged across multiple rounds. These are not one-time notes — they're documented patterns. If a note appears in Round 1 and again in Round 2, it will be noted.
10
Turnaround Standards Will Be Calibrated Per Style
Editing a 2–3 minute video in a detailed style (particularly Style 2) in 1–2 hours is a tight standard. Time expectations will be adjusted per style after completing each a few times. Communicate proactively if timing is a concern.

Style 2 — Analytical

Vibe: Clean, professional, data-driven. Think ESPN analytics segment — structured, smart, confident. Logic is visualized. Players are real, not illustrated. Color earns its place.

🎨 Visual Direction
✓ Do
Uniform, Consistent Background
Clean solid or minimal background. No crumpled paper, no texture, no busy patterns. The background should not compete with the data and graphics.
✗ Don't
No Text Animation on Body Text
Text animation on body content = too much. Static clean text only for body. Only use motion for graphic elements (player headshots, icons, data visuals) — not the words themselves.
✓ Do
Real Player Headshots
Use actual headshots — not animated clip art. Flagged in Round 1. If a headshot isn't available, find the closest real photo. Never use illustrated player figures.
✓ Do
Visual Wordplay on Player Names
When a player's name allows it, use literal visual wordplay. Example: Makai Lemon = player headshot with an actual lemon beside him. Creative but restrained.
✓ Do
Always Have Movement on Screen
Elements should always be animating in some way. Reference: 0:17 — move lemon/player up, text down to middle. Nothing should sit fully static on screen at any moment.
ℹ Note
Proportional, Restrained Graphics
No super-large graphics. Scale is proportional — elements should feel like they belong in a structured layout, not dominating the frame. Apply this logic consistently throughout.
📊 Data Visualization
ℹ Data Logic
Show Stats as Structured Math Formulas
When the script references a stat relationship, show it as a formula with a dividing line — like a math problem on a whiteboard. Not plain text, not a list.

🔤 Text Rules
✓ Do
Body Text = Lowercase
Default text is lowercase. No sentence case, no title case, no all caps for regular body copy. This is the baseline for Style 2.
✓ Do
Names, Stats, Places = First Letter Cap Only
Use title case (e.g., Makai Lemon, At #11) — not all-caps. Not MAKAI LEMON. Not AT #11. First letter capital on each word that's a proper noun or stat.
✗ Don't
No UHF Brand Colors in Text
Pink and blue are not default text colors. They appear only if they are contextually meaningful to the specific content — which is rare. Assume no UHF colors unless there is a specific reason.
ℹ Color Rule
Color Only When It Means Something
Very few scenes need color. Not every sentence. Use color only on words where the meaning matters.
Green = Good Red = Bad Yellow/Orange = Middle Ground
✗ Don't
Don't Color Player Names
Names stay white unless the name itself is the emphasis. Color on a name is only justified if the name is the point — not just because a player is being mentioned.

Style 3 — Cinematic

Vibe: Cinematic, high-impact, news/injury storytelling. Think documentary intro meets sports news broadcast. Dark, dramatic, purposeful. Every visual choice tells part of the story.

🎨 Visual Direction
✓ Do
Dark, Dramatic Background
The background should feel cinematic and moody. Dark tones. The kind of visual weight you see in injury reports or big game previews. Not plain black — textured darkness with depth.
✓ Do
Camera Pans the Whole Scene
For pan-out effects, the camera moves — not the graphics. The entire scene (players, text, graphics) moves together as if filmed. Individual elements do not scale down or shrink. Reference: Pan out.mp4.
✓ Do
Player Cutouts With Dramatic Framing
Use cutout style players — spotlight, silhouette, cinematic framing. Not headshots in boxes. The player should feel like they're part of the scene, not pasted on top.
✓ Do
Lineup Storytelling: Highlight vs. Shadow
Use lighting as narrative:
💡 Highlighted = Rising / Relevant 🌑 Shadowed = Downgraded / Out
Jersey swaps (old team → new team) always happen in the shadowed state — the faded/dimmed version, not the lit version.
✓ Do
Hook Is Always Your Best Work
The approved sample's 0:01–0:05 was specifically praised: "Very strong hook editing." The first 2–4 seconds need to be your best work every time. Ambulance should appear as early as possible. Don't delay the visual hit.
ℹ Note
Thematic Props and Graphics
Props like the ambulance are part of the storytelling — they set tone and context. Don't remove thematic elements unless there's a specific conflict with the transition. Communicate before removing.
🔤 Text Rules
✓ Do
80% of Video = White, Lowercase, No Emphasis
The default state is white, lowercase. Most sentences, most words, most of the video. Emphasis is rare and earned — not the baseline.
✓ Do
Caps + Color = Surgical Emphasis
Only use caps or color on specific key words where meaning needs to land hard. Not on full sentences. Not on names. Not every scene. Example:
had similar
but LESS
SEVERE
injuries
✓ Do
Names, Teams, Places = Proper Title Case
Proper nouns use title case: Giants not GIANTS. New York not NEW YORK. Title case for proper nouns — not all caps, not lowercase.
✗ Don't
No UHF Brand Colors in Text
Pink and blue stay out. This style uses yellow for caution/middle ground as the primary emphasis color. No UHF brand palette in captions or body text.
